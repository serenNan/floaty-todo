# 待办事项管理

## 目标文件解析

每次执行 `/todo` 前, 按以下优先级解析"目标文件":

1. **cwd 下存在 `TODO.md`** (Windows 文件系统大小写不敏感) → 用它
2. **cwd 下不存在 `TODO.md` 但 cwd 像项目目录** (含 `.git` / `package.json` / `Cargo.toml` / `pyproject.toml` / `go.mod` / `pom.xml` 之一) → 在 cwd 创建 `TODO.md` 写入四象限模板, 然后操作它
3. **否则 fallback 到全局** `D:/Notes/📅TODO.md`

只在 cwd 当前目录检测, 不递归向上找。解析完毕后, 后续"目标文件"统一指代解析结果。

检测命令示例 (PowerShell):
```powershell
Test-Path TODO.md                    # 第 1 步
Test-Path .git, package.json, Cargo.toml, pyproject.toml, go.mod, pom.xml | Where-Object { $_ }   # 第 2 步项目特征
```

## 文件结构

待办按四象限组织 (全局和项目本地结构一致):

```
## 🔴 紧急+重要
- [ ] 任务1

## 🟡 重要不紧急
- [ ] 任务2

## 🟠 紧急不重要
- [ ] 任务3

## 🟢 不紧急不重要
- [ ] 任务4
```

**全局文件** `D:/Notes/📅TODO.md` 末尾额外有归档区:
```
***
## 📦 归档
- [[TODO/archive|查看历史归档]]
```
归档文件 `D:/Notes/TODO/archive.md` 按 `### YYYY-MM-DD` 日期分组, 由页面顶部 dataviewjs 按钮自动管理(skill 只标记完成, 不动归档)。

**项目本地 `TODO.md`** 新建时只写四象限四个标题, **不写归档区** (项目里没 dataviewjs, wikilink 无意义)。完成的任务原位保留 `- [x] ✅ YYYY-MM-DD`, 由用户手动清理。

## 操作说明

用户输入：`$ARGUMENTS`

### 1. 添加待办

在**目标文件**对应象限标题下追加 `- [ ] 任务描述`

象限判断(艾森豪威尔矩阵):
- 🔴 紧急+重要: 有截止日期且影响大(线上 bug、客户催交、考试/答辩)
- 🟡 重要不紧急: 影响大但没有截止压力(技能提升、简历完善、架构优化)
- 🟠 紧急不重要: 有时间限制但可委托/快速处理(整理文件、修小 bug、清理缓存)
- 🟢 不紧急不重要: 有空再做(研究新工具、学习新框架)
- 不确定 -> 问用户

格式规则:
- 截止日期: 追加 `📅 YYYY-MM-DD`
- 提醒时间: 使用 Obsidian Reminder 格式 `(@YYYY-MM-DD HH:mm)`, 先用 `date "+%Y-%m-%d %H:%M"` 获取当前时间再计算相对时间
- 子任务/阶段: 用缩进 `\t- [ ]` 表示子任务
- 关联笔记用 `[[wikilink]]` 语法
- 任务描述尽量简短, 控制在一行以内

### 2. 查看待办

读取**目标文件**, 按象限展示未完成的 `- [ ]` 任务。

筛选模式(根据用户意图自动判断):
- **全部**: 展示所有未完成任务
- **今日到期**: 筛选 `📅` 日期 <= 今天的任务
- **逾期**: 筛选 `📅` 日期 < 今天的任务, 高亮提醒
- **某象限**: 只展示指定象限下的任务

### 3. 完成待办

将任务的 `- [ ]` 改为 `- [x]`, 并追加 `✅ YYYY-MM-DD`。

用 `date "+%Y-%m-%d"` 获取今天日期。

支持批量完成: 用户说"紧急的都完成了"时, 将该象限下所有任务标记完成。

归档:
- **目标文件 = 全局** → 由页面顶部 dataviewjs 按钮处理(自动移到 `TODO/archive.md`), skill 只负责标记完成, 不要手动操作 archive.md
- **目标文件 = 项目本地** → 没有归档机制, 完成的任务原位保留, 用户自行清理

### 4. 删除待办

直接从**目标文件**中删除对应行。

## 注意事项

- 每次操作前都要重新解析目标文件 (cwd 可能在不同 session 间变化)
- 项目本地和全局是两个独立的待办池, **不要跨文件迁移任务**
- 不要创建单独的任务文件 (除了根据规则在 cwd 创建的 `TODO.md`)
- 操作完成后, 在简短确认里**指明用的是哪个文件**, 避免用户误以为写错了池

## 输出

操作完成后简短确认, 附目标文件标识:
- 项目本地: `已添加到🔴紧急+重要 (项目本地 ./TODO.md)`
- 全局: `已添加到🔴紧急+重要 (全局)`
- 新建场景: `项目本地 TODO.md 不存在, 已新建并写入四象限模板`

查看时用简洁列表展示, 逾期任务标注"已逾期", 顶部注明读的是哪个文件。
